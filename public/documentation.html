Movie API Program Documentation This Movie API will integrate with a database storing movies (titles), their genre, director, image URLs, and more. By using this API, the user can obtain detailed information about directors (bio, birth year, death year), detailed information about specific genres, and the user can interact with the API to make a user profile. This profile will allow them to store a list of favorite movies they can add on to. The user will be able to modify their profile in multiple ways and also de-register from the platform entirely.
Endpoints at continued after 'https://node-movie-api-mattg.herokuapp.com/'
  • Return a list of ALL movies to the user. GET method, accessed by '/movies' URL. No parameters. Returns JSON list of all movies in the database in a template listed in the next documentation endpoint.
  • Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user. GET method, accessed by '/movies/:movieTitle' where ':movieTitle' is the title of the movie. See END of documentation page for full list of movies.

  • Return a list of ALL genres to the user. GET method, accessed by '/genres' URL. No parameters. Returns JSON list of genres as so '{ "Genre": "Thriller" "Description": "Action-packed and fast-paced with moments full of tension, anxiety, and fear" }'
  • Return data of the specified genre to the user. GET method, accessed by '/genres/:genreName' URL where ':name' parameter is the name of a genre. Returns JSON list of the specific genre as so '{ "Genre": "Thriller" "Description": "Action-packed and fast-paced with moments full of tension, anxiety, and fear" }'
  • Return list of all directors to the user. GET method, accessed by '/directors' URL. Returns JSON list of all directors as so '{ "Full Name": "Hayao Miyazaki" "Birth Year": "1941" }'
  • Return data of the specified director to the user. GET method, accessed by '/directors/:name' URL where ':name' parameter is the name of a director. Returns JSON list of the specific director as so '{ "Full Name": "Hayao Miyazaki" "Birth Year": "1941" }'
  • Allow new users to register via POST method. URL ends in '/users/' and is expecting a JSON input with fields from the user object template: "Username", "Password" "Email", "Birthday", and "FavoriteMovies"(array, FIELD NOT YET IMPLEMENTED).
    • Expects a JSON object as such:
      {
      "Username" : "jimothywilliams",
      "Password" : "smithjimmothy",
      "Email" : "jimmy@smith.net"
      }
    • Will return a JSON object response showing POSTed user, filled required fields, and non-empty optional fields (birthday, FavoriteMovies)
  • Allow new users to edit their info (e.g. username) via PUT method. URL ends in '/users/:name' where ':name' is the current username of the user.
    • Expects a JSON object as such:
      {
      "Username" : "jimothywilliams",
      "Password" : "smithjimmothy",
      "Email" : "jimmy@smith.net"
      }
  • Allow users to create a list of favorite movies via POST method to the URL ending '/user/lists/:name' where ':name' is the name of the list. (NOT YET IMPLEMENTED)
  • Allow users to update a list of favorite movies and ADD a movie to the list via PUT method to the URL ending '/user/lists/:name' where ':name' is the name of the list. (NOT YET IMPLEMENTED)
  • Allow users to update a list of favorite movies and REMOVE a movie to the list via DELETE method to the URL ending '/user/lists/:name' where ':name' is the name of the list. (NOT YET IMPLEMENTED)
  • Allow users to DE-register via DELETE method. URL ends in '/users/:Username' where ':name' is the username of the user. Expects a ':Username' parameter of a valid, registered username. Simple response stating the user is deleted, or the user was not found. (NOT YET IMPLEMENTED)
